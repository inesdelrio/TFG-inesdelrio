# RF34 - Mapa de eventos y entidades

## Estado del requisito

Cerrado y validado en entorno local.

## Descripcion funcional

Incorporar mapas dentro de VolunRed para visualizar ubicaciones de eventos y entidades dentro de Madrid, reutilizando las coordenadas guardadas en los modelos `Event` y `Entity`.

La funcionalidad permite consultar marcadores geograficos segun el rol del usuario y acceder desde cada marcador a la informacion principal o al detalle correspondiente.

## Alcance limitado a Madrid

El sistema esta limitado a la ciudad de Madrid. Las coordenadas se validan con un bounding box aproximado:

- latitud minima: `40.312`
- latitud maxima: `40.643`
- longitud minima: `-3.889`
- longitud maxima: `-3.517`

Las vistas de mapa tambien limitan la navegacion visual a Madrid mediante bounds de Leaflet.

## Comportamiento por rol

### Voluntario

- Ruta: `GET /eventos/mapa`.
- Muestra eventos activos, futuros, pertenecientes a entidades verificadas y con coordenadas.
- Cada marcador de evento muestra titulo, entidad organizadora, fecha, direccion y enlace al detalle publico.

### Entidad

- Ruta: `GET /entidad/mapa`.
- Muestra eventos propios con coordenadas.
- Muestra la ubicacion de la propia organizacion si tiene coordenadas.
- Los marcadores de evento y organizacion se distinguen visualmente.
- Cada evento enlaza al detalle del evento.
- El marcador de organizacion muestra nombre y direccion.

### Administrador

- Ruta: `GET /admin/mapa`.
- Muestra eventos con coordenadas y entidades con coordenadas.
- Eventos y entidades se distinguen visualmente.
- Cada marcador muestra informacion basica:
  - evento: titulo, entidad, fecha, estado y enlace al detalle;
  - entidad: nombre, estado, email o solicitante si procede y enlace al detalle administrativo.

## Busqueda de direcciones

La aplicacion permite buscar direcciones de Madrid mediante una busqueda explicita:

- el usuario escribe una direccion;
- pulsa `Buscar direccion`;
- el backend consulta Nominatim/OpenStreetMap;
- se muestran sugerencias;
- al seleccionar una sugerencia se rellenan direccion visible, `normalizedAddress`, `latitude` y `longitude`.

No se usa autocomplete continuo por cada tecla. Esto evita llamadas excesivas a Nominatim y mantiene un uso mas razonable del servicio externo.

## Validacion y almacenamiento

Los modelos `Event` y `Entity` almacenan:

- `latitude`
- `longitude`
- `normalizedAddress`

La utilidad `validateMadridLocation` comprueba latitud, longitud, valores numericos, coordenadas dentro de Madrid y direccion normalizada cuando se exige.

Los eventos requieren una direccion valida de Madrid para crearse o editarse. El perfil de entidad permite guardar la ubicacion de la organizacion con coordenadas validas.

## Tecnologias usadas

- Leaflet para renderizar mapas en frontend.
- OpenStreetMap como capa base.
- Nominatim/OpenStreetMap para busqueda explicita de direcciones.
- Prisma y PostgreSQL para almacenar coordenadas.
- EJS para renderizar vistas y pasar marcadores como JSON seguro.

## Limitaciones

- Requiere internet para consultar Nominatim, cargar Leaflet desde CDN y cargar teselas de OpenStreetMap.
- Nominatim no se usa como autocomplete continuo.
- No hay clustering de marcadores.
- El alcance geografico esta limitado a Madrid.
- Los eventos o entidades antiguos sin coordenadas no aparecen en los mapas hasta que se actualicen.
- Si OpenStreetMap o Nominatim no estan disponibles, la busqueda o visualizacion puede verse limitada.

## Comprobacion manual

1. Ejecutar `npm run dev`.
2. Iniciar sesion como entidad.
3. Abrir `http://localhost:3000/entidad/perfil`.
4. Buscar y seleccionar una direccion de Madrid.
5. Guardar el perfil.
6. Crear o editar un evento con direccion valida de Madrid.
7. Abrir `http://localhost:3000/entidad/mapa` y comprobar organizacion/eventos propios.
8. Iniciar sesion como voluntario y abrir `http://localhost:3000/eventos/mapa`.
9. Confirmar que aparecen eventos activos, futuros y verificados con coordenadas.
10. Iniciar sesion como administrador y abrir `http://localhost:3000/admin/mapa`.
11. Confirmar que se muestran eventos y entidades con coordenadas.
