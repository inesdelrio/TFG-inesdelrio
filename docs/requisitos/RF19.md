# RF19 - Historial de participacion

## Estado del requisito

Cerrado y validado en entorno local.

## Objetivo

Permitir que el voluntario consulte sus actividades inscritas, separando actividades futuras y actividades pasadas a partir de la fecha del evento.

## Alcance funcional

- El voluntario accede al historial desde `GET /voluntariado/historial`.
- La ruta queda protegida con sesion activa y rol `VOLUNTARIO`.
- El historial muestra actividades futuras y pasadas en secciones separadas.
- Cada actividad muestra:
  - titulo del evento
  - entidad organizadora
  - fecha y hora
  - ciudad y direccion
  - estado de publicacion
- Los eventos activos enlazan al detalle publico.
- Los eventos retirados o no visibles muestran un aviso y no dejan un enlace publico activo.

## Implementacion realizada

### 1. Servicio de historial

- Se ha creado `src/services/volunteers/get-volunteer-history.service.js`.
- El servicio consulta `EventRegistration` filtrando por `volunteerUserId`.
- La consulta incluye el evento y la entidad organizadora mediante la relacion `EventRegistration -> Event -> Entity`.
- Las inscripciones se transforman a un modelo de vista con `futureActivities` y `pastActivities`.

### 2. Ruta y controlador

- Se ha creado `src/controllers/volunteers/volunteer-history.controller.js`.
- Se ha anadido la ruta `GET /voluntariado/historial` en `src/routes/volunteers/index.js`.
- La ruta reutiliza `requireAuth` y `requireRole("VOLUNTARIO")`.

### 3. Vista y navegacion

- Se ha creado `src/views/pages/volunteers/history.ejs`.
- Se ha anadido acceso desde el area del voluntario.
- Se ha anadido acceso en la navegacion superior para usuarios con rol `VOLUNTARIO`.

## Regla sobre cancelaciones

El diseno actual de `RF13` elimina la inscripcion cuando el voluntario cancela una actividad. Por tanto, `RF19` muestra las inscripciones existentes en la base de datos. La conservacion de cancelaciones historicas queda como linea futura y requeriria un cambio de modelo o un estado de inscripcion.

## Migraciones y base de datos

- `RF19` no requiere cambios en `prisma/schema.prisma`.
- No se ha creado ninguna migracion.
- Se reutiliza el modelo existente `EventRegistration`.

## Comprobacion manual

1. Ejecutar `npm run dev`.
2. Iniciar sesion con un usuario voluntario.
3. Inscribirse en una o varias actividades.
4. Abrir `http://localhost:3000/voluntariado/historial`.
5. Comprobar que las actividades aparecen separadas entre futuras y pasadas.
6. Confirmar que los eventos activos permiten abrir el detalle.
7. Confirmar que los eventos retirados muestran el estado y no ofrecen enlace publico activo.
