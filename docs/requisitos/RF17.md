# RF17 - Panel principal de administracion

## Estado del requisito

Cerrado y validado en entorno local.

## Objetivo

Crear el area base de administracion de la plataforma con un panel resumen y accesos directos a entidades, usuarios y publicaciones.

## Alcance funcional

- Existe un panel principal en `GET /admin/area`.
- El panel muestra metricas simples del sistema.
- El administrador dispone de accesos directos a:
  - entidades
  - usuarios
  - publicaciones
- Todas las rutas del panel quedan protegidas para el rol `ADMIN`.

## Implementacion realizada

### 1. Rutas protegidas del panel

- `GET /admin/area` muestra el panel principal.
- `GET /admin/entidades` mantiene el listado de entidades pendientes.
- `GET /admin/usuarios` muestra las cuentas registradas.
- `GET /admin/publicaciones` muestra los eventos publicados.
- Todas estas rutas reutilizan `requireAuth` y `requireRole("ADMIN")`.

### 2. Resumen y metricas

- Se ha creado `src/services/admin/get-admin-dashboard-summary.service.js`.
- El servicio obtiene contadores basicos de:
  - usuarios totales
  - voluntarios
  - entidades totales
  - entidades pendientes
  - entidades verificadas
  - eventos totales
  - eventos futuros

### 3. Vistas del area administrativa

- Se ha creado `src/views/pages/admin/dashboard.ejs` como layout principal del panel.
- Se ha creado `src/views/pages/admin/users-list.ejs` para usuarios.
- Se ha creado `src/views/pages/admin/publications-list.ejs` para publicaciones.
- Se ha ajustado `src/views/pages/admin/entities-list.ejs` para integrarlo como seccion del panel y enlazar con el resto de areas.

### 4. Navegacion interna

- Desde `/admin/area` el administrador puede acceder a entidades, usuarios y publicaciones.
- La seccion de entidades conserva el flujo de revision creado en `RF04`.
- El panel queda conectado a la redireccion por rol existente hacia `/admin/area`.

### 5. Tests unitarios

- Se ha anadido un test unitario para el servicio de resumen del panel administrativo.
- El test comprueba que las metricas basicas se calculan y devuelven correctamente.

## Migraciones y base de datos

- No ha sido necesaria ninguna migracion nueva.

## Comprobacion manual

1. Ejecutar `npm run dev`.
2. Iniciar sesion con una cuenta de administrador.
3. Abrir `http://localhost:3000/admin/area`.
4. Verificar la carga de metricas basicas.
5. Acceder a `Entidades`, `Usuarios` y `Publicaciones`.
6. Confirmar que cada seccion carga correctamente.
7. Intentar acceder con un usuario no administrador y comprobar la restriccion por rol.
