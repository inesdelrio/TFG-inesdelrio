# RF21 - Baja definitiva de cuenta

## Estado del requisito

Cerrado y validado en entorno local.

## Descripcion

Permitir que un usuario con rol `VOLUNTARIO` o `ENTIDAD` solicite la baja definitiva de su cuenta, aplicando anonimizacion y limpieza de relaciones futuras sin romper datos historicos necesarios para la coherencia del sistema.

## Criterio de validacion

- Un voluntario puede acceder a `GET /voluntariado/eliminar-cuenta`.
- Una entidad puede acceder a `GET /entidad/eliminar-cuenta`.
- La baja exige confirmacion explicita.
- La cuenta queda anonimizada.
- Las sesiones antiguas de usuarios anonimizados quedan bloqueadas.
- Las relaciones futuras se limpian segun el rol.
- No se realiza borrado fisico directo del usuario principal.

## Alcance funcional

- Baja de voluntario.
- Baja de entidad.
- Confirmacion previa mediante formulario.
- Anonimizacion de datos de usuario.
- Limpieza de inscripciones futuras, seguimientos y notificaciones afectadas.
- Retirada de eventos futuros de una entidad eliminada.
- Conservacion controlada de informacion historica.

## Implementacion tecnica

- `src/routes/volunteers/index.js` define las rutas de baja de voluntario.
- `src/routes/entities/index.js` define las rutas de baja de entidad.
- `src/controllers/accounts/account-deletion.controller.js` renderiza el formulario y ejecuta la baja.
- `src/services/accounts/account-anonymization.js` construye los datos anonimizados.
- `src/services/accounts/delete-volunteer-account.service.js` anonimiza voluntarios y elimina relaciones futuras.
- `src/services/accounts/delete-entity-account.service.js` anonimiza entidades, retira eventos futuros y limpia relaciones afectadas.
- `src/views/pages/accounts/delete-confirmation.ejs` muestra la confirmacion.
- `src/middlewares/auth.middleware.js` refresca el usuario activo y bloquea sesiones de usuarios anonimizados.

## Comprobacion manual

1. Ejecutar `npm run dev`.
2. Iniciar sesion como voluntario.
3. Abrir `http://localhost:3000/voluntariado/eliminar-cuenta`.
4. Enviar la confirmacion requerida.
5. Comprobar que se destruye la sesion y que la cuenta no puede seguir usandose.
6. Repetir el flujo con una cuenta de entidad desde `http://localhost:3000/entidad/eliminar-cuenta`.
7. Confirmar que los eventos futuros de la entidad quedan retirados y que las relaciones futuras se limpian.
