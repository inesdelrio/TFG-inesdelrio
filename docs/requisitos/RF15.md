# RF15 - Notificaciones internas

## Estado del requisito

Cerrado y validado en entorno local.

## Objetivo

Generar avisos internos para usuarios autenticados con rol `VOLUNTARIO` o `ENTIDAD`, permitiendo consultar el listado propio, abrir el detalle, marcar avisos como leidos o no leidos y borrar notificaciones propias.

## Alcance funcional

- Existe un modelo de notificacion interna persistido en Prisma.
- Cada notificacion tiene un destinatario general mediante `recipientUserId`.
- Cada notificacion puede guardar el usuario que origina la accion mediante `actorUserId`.
- Al publicar o modificar un evento se generan notificaciones para los voluntarios suscritos a la entidad.
- Al inscribirse un voluntario, seguir una entidad o completarse el aforo de un evento se generan notificaciones para la entidad propietaria.
- Voluntarios y entidades disponen de un listado propio de notificaciones en `GET /notificaciones`.
- Cada usuario puede consultar el detalle propio en `GET /notificaciones/:notificationId`.
- Cada notificacion propia puede marcarse como leida o no leida.
- Cada notificacion propia puede borrarse.
- El sistema bloquea el acceso a notificaciones ajenas.

## Implementacion realizada

### 1. Modelo y migracion

- Se ha definido el modelo `InternalNotification` en Prisma.
- La notificacion queda vinculada a:
  - usuario destinatario mediante `recipientUserId`
  - usuario actor opcional mediante `actorUserId`
  - entidad relacionada
  - evento relacionado cuando aplica
- Se guarda tipo, titulo, mensaje, estado de lectura y fecha de creacion.

### 2. Generacion de notificaciones

- Se ha creado `src/services/notifications/create-event-notifications.service.js`.
- El servicio recupera la lista de voluntarios suscritos a la entidad.
- Cuando una entidad publica un evento se generan notificaciones con tipo `EVENT_PUBLISHED`.
- Cuando una entidad modifica un evento se generan notificaciones con tipo `EVENT_UPDATED`.
- Si la entidad no tiene suscriptores, no se crean registros.
- Se ha creado `src/services/notifications/create-entity-registration-notification.service.js` para avisar a la entidad cuando un voluntario se inscribe en un evento propio.
- Se ha creado `src/services/notifications/create-entity-followed-notification.service.js` para avisar a la entidad cuando un voluntario empieza a seguirla.
- Se ha creado `src/services/notifications/create-event-full-notification.service.js` para avisar a la entidad cuando un evento completa sus plazas.

### 3. Integracion con eventos

- `src/services/events/create-event.service.js` genera avisos despues de publicar el evento.
- `src/services/events/update-owned-event.service.js` genera avisos despues de editar el evento propio.
- El flujo reutiliza la relacion de suscripcion creada en `RF11`.

### 4. Consulta, detalle y acciones

- Se ha creado `src/services/notifications/list-user-notifications.service.js`.
- Se ha creado `src/services/notifications/get-notification-detail.service.js`.
- Se ha creado `src/services/notifications/mark-notification-as-read.service.js`.
- Se ha creado `src/services/notifications/set-notification-read-status.service.js`.
- Se ha creado `src/services/notifications/delete-notification.service.js`.
- Se ha creado `src/controllers/notifications/notification.controller.js`.
- Se ha creado la ruta protegida `GET /notificaciones`.
- Se ha creado la ruta protegida `GET /notificaciones/:notificationId`.
- Se ha creado la ruta protegida `POST /notificaciones/:notificationId/leida`.
- Se ha creado la ruta protegida `POST /notificaciones/:notificationId/no-leida`.
- Se ha creado la ruta protegida `POST /notificaciones/:notificationId/abrir`.
- Se ha creado la ruta protegida `POST /notificaciones/:notificationId/eliminar`.
- Las rutas exigen sesion activa y rol `VOLUNTARIO` o `ENTIDAD`.

### 5. Vista e integracion en la interfaz

- Se ha creado `src/views/pages/notifications/index.ejs`.
- Se ha creado `src/views/pages/notifications/detail.ejs`.
- La cabecera del sitio incorpora el acceso `Notificaciones` para usuarios con rol `VOLUNTARIO` y `ENTIDAD`.

  - actor relacionado cuando aplica
  - fecha de generacion
  - enlace al evento relacionado cuando existe
  - accion para marcar como leida o no leida la notificacion
  - accion para borrar la notificacion

## Comprobacion manual

1. Ejecutar `npm run dev`.
2. Iniciar sesion con una cuenta de voluntario y suscribirse a una entidad.
3. Iniciar sesion con la entidad seguida y publicar un evento nuevo.
4. Volver a entrar con el voluntario y abrir `http://localhost:3000/notificaciones`.
5. Verificar que aparece el aviso de nueva actividad.
6. Editar el evento desde la cuenta de entidad y comprobar el aviso de actividad actualizada.
7. Pulsar `Marcar como leida` y despues `Marcar como no leida`.
8. Abrir el detalle de la notificacion.
9. Borrar la notificacion y confirmar que desaparece del listado.
10. Iniciar sesion como entidad y generar una inscripcion o seguimiento desde un voluntario.
11. Comprobar que la entidad recibe su notificacion propia.
