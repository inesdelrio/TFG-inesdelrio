# RF23 - Mejora de estado y navegacion en inscripciones

## Estado del requisito

Cerrado y validado en entorno local.

## Descripcion

Mejorar la experiencia del voluntario al inscribirse o cancelar una inscripcion, mostrando estados claros en el detalle del evento y manteniendo una navegacion coherente al volver al listado de eventos.

## Criterio de validacion

- Tras inscribirse, el usuario vuelve al detalle del evento con mensaje de confirmacion.
- Tras cancelar, el usuario vuelve al detalle del evento con mensaje de confirmacion.
- El detalle muestra si el voluntario ya esta inscrito.
- El detalle muestra si el evento esta completo.
- Los casos de duplicado, evento completo o evento inactivo redirigen con estado controlado.
- El boton atras del navegador puede devolver al listado cuando el flujo viene de eventos.

## Alcance funcional

- Mensajes de estado mediante query string.
- Redireccion controlada al detalle del evento.
- Estado visual para inscripcion existente.
- Estado visual para evento completo.
- Manejo de errores funcionales esperados.
- Mejora de navegacion con `backToEvents`.

## Implementacion tecnica

- `src/controllers/events/event-registration.controller.js` resuelve la ruta de retorno y anade flags como `registered`, `cancelled`, `alreadyRegistered`, `full`, `inactive` y `backToEvents`.
- `src/controllers/events/event-publication.controller.js` interpreta esos flags para renderizar mensajes y configurar la navegacion.
- `src/services/events/get-event-registration-status.service.js` comprueba si el voluntario ya esta inscrito.
- `src/services/events/create-event-registration.service.js` valida duplicados, aforo, estado del evento y entidad verificada.
- `src/services/events/delete-event-registration.service.js` elimina la inscripcion propia y gestiona el caso de no inscrito.
- `src/views/pages/events/detail.ejs` muestra estados, botones y script de retorno al listado.

## Comprobacion manual

1. Ejecutar `npm run dev`.
2. Iniciar sesion como voluntario.
3. Abrir un evento activo desde `http://localhost:3000/eventos`.
4. Pulsar `Apuntarse`.
5. Verificar el mensaje de inscripcion y el estado `Ya estas inscrito`.
6. Pulsar `Cancelar inscripcion`.
7. Verificar el mensaje de cancelacion.
8. Probar un evento completo y confirmar que no permite inscribirse.
9. Usar el boton atras tras una accion y comprobar el retorno al listado de eventos.
