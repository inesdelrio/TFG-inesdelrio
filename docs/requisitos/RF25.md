# RF25 - Mejora de notificaciones internas

## Estado del requisito

Cerrado y validado en entorno local.

## Descripcion

Ampliar el sistema de notificaciones internas para cubrir acciones relevantes de entidades y voluntarios, anadir detalle propio, permitir borrar notificaciones y mejorar la navegacion desde los avisos.

## Criterio de validacion

- Las entidades reciben notificaciones cuando un voluntario se inscribe en un evento propio.
- Las entidades reciben notificaciones cuando un voluntario empieza a seguirlas.
- Las entidades reciben notificaciones cuando un evento completa sus plazas.
- Los voluntarios siguen recibiendo notificaciones por eventos publicados o actualizados por entidades seguidas.
- El usuario autenticado puede abrir el detalle de una notificacion propia.
- El usuario autenticado puede marcar como leida o no leida una notificacion propia.
- El usuario autenticado puede borrar una notificacion propia.
- El sistema rechaza operaciones sobre notificaciones ajenas.

## Alcance funcional

- Generalizacion de notificaciones para `VOLUNTARIO` y `ENTIDAD`.
- Identificacion del destinatario con `recipientUserId`.
- Identificacion del actor opcional con `actorUserId`.
- Detalle de notificacion con datos del actor y evento cuando aplican.
- Borrado de notificaciones desde listado y detalle.
- Navegacion desde una notificacion hacia el detalle del evento relacionado.

## Implementacion tecnica

- `src/services/notifications/create-entity-registration-notification.service.js` crea avisos de nueva inscripcion para la entidad.
- `src/services/notifications/create-entity-followed-notification.service.js` crea avisos de nuevo seguimiento para la entidad.
- `src/services/notifications/create-event-full-notification.service.js` crea avisos cuando un evento alcanza el aforo completo.
- `src/services/notifications/get-notification-detail.service.js` carga el detalle propio con actor y evento.
- `src/services/notifications/delete-notification.service.js` elimina notificaciones propias.
- `src/services/notifications/set-notification-read-status.service.js` permite marcar como no leida.
- `src/controllers/notifications/notification.controller.js` conecta las acciones de detalle, lectura, no lectura, apertura y borrado.
- `src/views/pages/notifications/index.ejs` y `src/views/pages/notifications/detail.ejs` presentan las acciones en la interfaz.

## Comprobacion manual

1. Ejecutar `npm run dev`.
2. Iniciar sesion como voluntario y seguir una entidad.
3. Iniciar sesion como entidad y publicar un evento.
4. Confirmar que el voluntario recibe la notificacion.
5. Inscribir un voluntario en un evento de la entidad.
6. Confirmar que la entidad recibe la notificacion con datos del voluntario en el detalle.
7. Completar las plazas del evento y confirmar la notificacion de aforo completo.
8. Marcar una notificacion como leida y luego como no leida.
9. Borrar una notificacion desde el listado o desde el detalle.
