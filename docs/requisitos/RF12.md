# RF12 - Inscripcion en actividades

## Estado del requisito

Cerrado y validado en entorno local.

## Objetivo

Permitir que un voluntario se apunte a una actividad publicada mientras existan plazas disponibles, evitando duplicados y reflejando en la interfaz la disponibilidad real.

## Alcance funcional

- Existe relacion `voluntario-evento` en base de datos.
- La inscripcion se realiza desde la ficha publica del evento.
- Solo usuarios con rol `VOLUNTARIO` pueden apuntarse.
- No se permiten inscripciones duplicadas.
- La inscripcion se bloquea si el evento ya no esta activo o no quedan plazas.
- La ficha muestra plazas disponibles e informa del estado de inscripcion.

## Implementacion realizada

### 1. Modelo de datos

- Se ha creado el modelo `EventRegistration` en Prisma.
- La relacion une `User` con `Event`.
- Se ha añadido restriccion `@@unique([volunteerUserId, eventId])`.
- Se han añadido relaciones auxiliares en `User` y `Event`.

### 2. Migracion aplicada

- Se ha generado y aplicado la migracion `20260420120337_add_event_registration_rf12`.
- La migracion crea la tabla de inscripciones y la restriccion unique para evitar duplicados.

### 3. Reglas de negocio

- Se ha creado el servicio `src/services/events/create-event-registration.service.js`.
- El servicio comprueba que:
  - el evento existe
  - el evento sigue activo
  - no existe ya inscripcion previa del voluntario
  - siguen quedando plazas disponibles
- La disponibilidad se calcula con `totalSlots - registrations`.

### 4. Interfaz y ruta

- Se ha creado la ruta `POST /eventos/:eventId/inscribirse`.
- La ruta requiere sesion activa y rol `VOLUNTARIO`.
- Se ha añadido el boton `Apuntarse` en `src/views/pages/events/detail.ejs`.
- Si ya existe inscripcion o no quedan plazas, la interfaz muestra el estado correspondiente.

### 5. Estado de inscripcion

- Se ha creado `src/services/events/get-event-registration-status.service.js`.
- El detalle del evento consulta si el voluntario ya esta inscrito.
- La ficha muestra:
  - plazas totales
  - plazas disponibles
  - boton de inscripcion o estado actual del usuario

### 6. Tests unitarios

- Se ha añadido un test unitario de inscripcion correcta.
- Se ha añadido un test unitario de duplicado bloqueado.
- Se ha añadido un test unitario de evento completo bloqueado.
- Se ha añadido un test unitario del estado de inscripcion.

## Validacion realizada

- `npm run prisma:validate` ejecutado correctamente.
- `npm test` ejecutado correctamente.
- `POST /eventos/:eventId/inscribirse` crea la inscripcion cuando hay plazas.
- La segunda inscripcion del mismo voluntario no se duplica.
- La ficha del evento actualiza el estado de inscripcion y las plazas disponibles.

## Comprobacion manual

1. Ejecutar `npm run dev`.
2. Iniciar sesion con una cuenta de voluntario.
3. Abrir la ficha de un evento activo con plazas.
4. Pulsar `Apuntarse`.
5. Verificar la redireccion al detalle del evento con confirmacion.
6. Recargar la ficha y comprobar que ya no aparece el boton, sino el estado de inscripcion.
7. Verificar la reduccion de plazas disponibles.

## Evidencia de cierre

- Flujo HTTP validado en local con un voluntario temporal y un evento temporal.
- La inscripcion se ha guardado correctamente en PostgreSQL.
- El intento repetido no genera un segundo registro y la interfaz refleja que el voluntario ya esta inscrito.
