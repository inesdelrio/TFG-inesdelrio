# RF24 - Cancelacion de seguimiento de entidades

## Estado del requisito

Cerrado y validado en entorno local.

## Descripcion

Permitir que un voluntario deje de seguir una entidad desde el detalle de un evento, eliminando la relacion de seguimiento y actualizando el estado de la interfaz.

## Criterio de validacion

- Un voluntario puede seguir una entidad desde el detalle de un evento.
- Si ya sigue la entidad, ve el estado `Ya sigues a esta entidad`.
- Puede ejecutar `POST /entidades/:entityId/dejar-de-seguir`.
- La relacion `EntitySubscription` se elimina.
- Si no existe seguimiento, la operacion no rompe el flujo.
- La pantalla vuelve al detalle del evento con mensaje de confirmacion.

## Alcance funcional

- Cancelacion de seguimiento para voluntarios.
- Estado visual de seguimiento en el detalle de evento.
- Redireccion controlada al origen.
- Limpieza de la notificacion de seguimiento enviada a la entidad cuando corresponde.
- No incluye una pagina independiente de entidades seguidas.

## Implementacion tecnica

- `src/routes/entities/index.js` define `POST /entidades/:entityId/dejar-de-seguir`.
- `src/controllers/entities/entity-subscription.controller.js` ejecuta la baja de seguimiento y redirige con `unsubscribed=1`.
- `src/services/entities/delete-entity-subscription.service.js` elimina la relacion `EntitySubscription`.
- El mismo servicio borra la notificacion `ENTITY_FOLLOWED` asociada cuando existe.
- `src/services/entities/get-entity-subscription-status.service.js` permite mostrar el estado actual de seguimiento.
- `src/views/pages/events/detail.ejs` muestra los botones `Seguir entidad` o `Dejar de seguir`.

## Comprobacion manual

1. Ejecutar `npm run dev`.
2. Iniciar sesion como voluntario.
3. Abrir el detalle de un evento de una entidad verificada.
4. Pulsar `Seguir entidad`.
5. Confirmar que aparece el estado de seguimiento.
6. Pulsar `Dejar de seguir`.
7. Confirmar que el mensaje indica la cancelacion y vuelve a aparecer la opcion de seguir.
