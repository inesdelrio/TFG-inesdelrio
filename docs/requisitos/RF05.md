# RF05 - Perfil de voluntario

## Estado del requisito

Cerrado y validado en entorno local.

## Objetivo

Permitir que un voluntario autenticado consulte sus datos actuales y actualice los campos personales autorizados desde su area privada.

## Alcance funcional

- El voluntario puede abrir su perfil en `GET /voluntariado/perfil`.
- El formulario carga los datos actuales guardados en base de datos.
- Los campos editables en esta fase son `firstName`, `lastName` y `phone`.
- El `email` se muestra como referencia de acceso, pero no se edita desde `RF05`.
- La actualizacion se procesa en `POST /voluntariado/perfil`.
- Tras guardar, la sesion se sincroniza para reflejar el nuevo nombre visible.

## Implementacion realizada

### 1. Rutas y acceso

- Se ha creado el router `src/routes/volunteers/index.js`.
- Se han añadido las rutas `GET /voluntariado/perfil` y `POST /voluntariado/perfil`.
- Ambas rutas requieren sesion activa y rol `VOLUNTARIO`.

### 2. Controlador y carga de datos

- Se ha creado `src/controllers/volunteers/volunteer-profile.controller.js`.
- El controlador consulta el usuario autenticado y precarga nombre, apellidos, email y telefono.
- Si la actualizacion se completa, redirige a `/voluntariado/perfil?updated=1`.

### 3. Validacion y persistencia

- Se ha creado el validador `src/validators/volunteers/volunteer-profile.validator.js`.
- Se valida que `firstName` y `lastName` tengan al menos 2 caracteres.
- Se valida que `phone`, si se informa, tenga un formato basico valido.
- Se ha creado el servicio `src/services/volunteers/update-volunteer-profile.service.js`.
- El servicio actualiza solo los campos permitidos y mantiene el resto sin cambios.

### 4. Interfaz

- Se ha creado la vista `src/views/pages/volunteers/profile.ejs`.
- La pantalla muestra un resumen del perfil y el formulario editable.
- Se han añadido estilos especificos para la maquetacion del perfil.
- La navegacion del voluntario incluye acceso directo a `Perfil`.

### 5. Tests unitarios

- Se ha añadido un test unitario del servicio de actualizacion del perfil.
- Se ha añadido un test unitario del validador para datos invalidos.
- Ambos tests quedan integrados en `npm test`.

## Migraciones y base de datos

- `RF05` no requiere cambios de esquema en Prisma.
- No ha sido necesaria ninguna migracion nueva porque los campos usados ya existian en `User`.

## Validacion realizada

- `npm run prisma:validate` ejecutado correctamente.
- `npm test` ejecutado correctamente.
- Login real de voluntario comprobado correctamente.
- `GET /voluntariado/perfil` responde `200` con sesion valida.
- `POST /voluntariado/perfil` responde `302` a `/voluntariado/perfil?updated=1`.
- En base de datos, los cambios de nombre, apellidos y telefono quedan persistidos correctamente.
- La actualizacion mantiene la sesion coherente para mostrar el nuevo nombre visible.

## Comprobacion manual

1. Ejecutar `npm run dev`.
2. Iniciar sesion con una cuenta de voluntario.
3. Abrir `http://localhost:3000/voluntariado/perfil`.
4. Cambiar nombre, apellidos o telefono.
5. Guardar el formulario.
6. Verificar la redireccion a `?updated=1`.
7. Confirmar que el nuevo nombre aparece en el area de voluntario y en la navegacion.
