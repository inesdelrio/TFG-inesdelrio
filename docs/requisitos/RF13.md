# RF13 - Cancelacion de inscripcion

## Estado del requisito

Cerrado y validado en entorno local.

## Objetivo

Permitir que un voluntario anule su participacion en una actividad, eliminando su inscripcion y liberando automaticamente la plaza para otros usuarios.

## Alcance funcional

- El voluntario puede cancelar su inscripcion desde la ficha del evento.
- La cancelacion elimina la relacion `EventRegistration`.
- La plaza vuelve a quedar disponible en la interfaz.
- Si no existe inscripcion activa, la operacion se controla sin fallo.

## Implementacion realizada

### 1. Ruta y controlador

- Se ha creado la ruta `POST /eventos/:eventId/cancelar-inscripcion`.
- La ruta requiere sesion activa y rol `VOLUNTARIO`.
- Se ha ampliado `src/controllers/events/event-registration.controller.js` con la accion de cancelacion.

### 2. Logica de cancelacion

- Se ha creado `src/services/events/delete-event-registration.service.js`.
- El servicio busca la inscripcion por la clave compuesta `volunteerUserId_eventId`.
- Si existe, elimina el registro.
- Si no existe, devuelve error controlado `EVENT_REGISTRATION_NOT_FOUND`.

### 3. Liberacion de plazas

- No se mantiene un contador duplicado de plazas ocupadas.
- La disponibilidad ya se calcula con `totalSlots - registrations`.
- Al borrar la inscripcion, la plaza queda liberada automaticamente en base de datos y en la interfaz.

### 4. Interfaz

- Se ha añadido la accion `Cancelar inscripcion` en `src/views/pages/events/detail.ejs`.
- Cuando el usuario ya esta inscrito, se muestra el estado actual y el boton de cancelacion.
- Tras cancelar, el detalle del evento vuelve a mostrar las plazas disponibles actualizadas.

### 5. Calendario futuro

- Aunque el calendario personal aun no existe, la cancelacion deja actualizada la fuente de datos `EventRegistration`.
- Esto deja preparada la sincronizacion natural con `RF16`, ya que el calendario podra construirse a partir de las inscripciones vigentes.

### 6. Tests unitarios

- Se ha añadido un test unitario de cancelacion correcta.
- Se ha añadido un test unitario del caso en que no existe inscripcion activa.

## Migraciones y base de datos

- `RF13` no requiere cambios de esquema en Prisma.
- No ha sido necesaria ninguna migracion nueva porque reutiliza `EventRegistration` creado en `RF12`.

## Validacion realizada

- `npm run prisma:validate` ejecutado correctamente.
- `npm test` ejecutado correctamente.
- `POST /eventos/:eventId/cancelar-inscripcion` elimina la inscripcion correctamente.
- La plaza vuelve a quedar disponible en la ficha del evento.
- El intento de cancelar sin inscripcion previa se resuelve con mensaje controlado.

## Comprobacion manual

1. Ejecutar `npm run dev`.
2. Iniciar sesion con una cuenta de voluntario ya inscrita en un evento.
3. Abrir la ficha de la actividad.
4. Pulsar `Cancelar inscripcion`.
5. Verificar la redireccion al detalle con confirmacion.
6. Comprobar que desaparece el estado de inscrito y reaparece la opcion de apuntarse.
7. Verificar que las plazas disponibles aumentan.

## Evidencia de cierre

- Flujo HTTP validado en local con un voluntario temporal inscrito en un evento.
- La cancelacion ha eliminado correctamente la relacion en PostgreSQL.
- La ficha ha pasado de mostrar usuario inscrito a mostrar de nuevo la opcion de apuntarse.
