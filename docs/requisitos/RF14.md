# RF14 - Consulta de inscritos por parte de la entidad

## Estado del requisito

Cerrado y validado en entorno local.

## Objetivo

Permitir que la entidad autora de un evento consulte el listado de voluntarios apuntados a sus actividades, mostrando los datos basicos permitidos para la gestion del evento.

## Alcance funcional

- Existe listado de inscritos en `GET /eventos/:eventId/inscritos`.
- Solo la entidad autora puede acceder a ese listado.
- La vista muestra los datos basicos del voluntario inscrito.
- El acceso se integra desde la gestion de eventos propios.

## Implementacion realizada

### 1. Ruta y control de acceso

- Se ha creado la ruta protegida `GET /eventos/:eventId/inscritos`.
- La ruta requiere sesion activa y rol `ENTIDAD`.
- El control de propiedad reutiliza `getOwnedEventById` para asegurar que solo la entidad autora accede.

### 2. Consulta de inscritos

- Se ha creado `src/services/events/list-owned-event-registrations.service.js`.
- El servicio recupera el evento propio y sus inscripciones ordenadas por fecha.
- Cada inscripcion carga los datos basicos del voluntario:
  - nombre
  - apellidos
  - email
  - telefono

### 3. Controlador y vista

- Se ha creado `src/controllers/events/event-attendance.controller.js`.
- Se ha creado la vista `src/views/pages/events/attendees.ejs`.
- La pantalla muestra datos resumen del evento y el listado de inscritos.
- Si no hay inscritos, la interfaz muestra un estado vacio claro.

### 4. Integracion en la interfaz

- El listado de eventos propios incorpora el acceso `Inscritos`.
- La entidad puede ir desde el listado general de eventos a la gestion de asistentes de cada publicacion propia.

### 5. Tests unitarios

- Se ha añadido un test unitario de consulta correcta de inscritos con datos del voluntario.
- Se ha añadido un test unitario de control de propiedad para impedir acceso de entidades ajenas.

## Migraciones y base de datos

- `RF14` no requiere cambios de esquema en Prisma.
- No ha sido necesaria ninguna migracion nueva porque reutiliza `EventRegistration` de `RF12`.

## Validacion realizada

- `npm run prisma:validate` ejecutado correctamente.
- `npm test` ejecutado correctamente.
- `GET /eventos/:eventId/inscritos` responde `200` para la entidad autora.
- La vista carga correctamente los voluntarios inscritos.
- El acceso queda bloqueado cuando el evento no pertenece a la entidad autenticada.

## Comprobacion manual

1. Ejecutar `npm run dev`.
2. Iniciar sesion con una cuenta de entidad que tenga un evento con voluntarios inscritos.
3. Abrir `http://localhost:3000/eventos`.
4. Pulsar `Inscritos` sobre un evento propio.
5. Verificar la carga del listado de voluntarios.
6. Comprobar nombre, email y telefono de los inscritos.

## Evidencia de cierre

- Flujo HTTP validado en local con una entidad temporal, un evento temporal y varios voluntarios inscritos.
- La vista de inscritos ha respondido correctamente con los datos basicos esperados.
- El control de propiedad del evento queda cubierto en servicio y en tests unitarios.
