# RF06 - Perfil de entidad

## Estado del requisito

Cerrado y validado en entorno local.

## Objetivo

Permitir que una cuenta con rol `ENTIDAD` consulte y actualice la informacion visible de su perfil institucional sin modificar los datos identificativos de alta.

## Alcance funcional

- La entidad puede abrir su perfil en `GET /entidad/perfil`.
- El formulario carga la informacion actual almacenada en base de datos.
- Los campos editables en esta fase son:
  - `organizationName`
  - `contactEmail`
  - `contactPhone`
  - `city`
  - `address`
  - `description`
  - `supportingInfo`
- `legalName` y `taxId` se muestran como referencia y quedan bloqueados en esta version.
- La actualizacion se procesa en `POST /entidad/perfil`.

## Implementacion realizada

### 1. Rutas y acceso

- Se han añadido las rutas `GET /entidad/perfil` y `POST /entidad/perfil` en `src/routes/entities/index.js`.
- Ambas rutas requieren sesion activa y rol `ENTIDAD`.

### 2. Controlador y carga de datos

- Se ha creado `src/controllers/entities/entity-profile.controller.js`.
- El controlador recupera la entidad vinculada al usuario autenticado mediante `requestedByUserId`.
- Si el guardado es correcto, redirige a `/entidad/perfil?updated=1`.

### 3. Validacion y persistencia

- Se ha creado el validador `src/validators/entities/entity-profile.validator.js`.
- Se valida nombre visible, email de contacto, telefono, ciudad, direccion y descripcion.
- Se ha creado el servicio `src/services/entities/update-entity-profile.service.js`.
- El servicio actualiza solo los campos institucionales permitidos.

### 4. Interfaz

- Se ha creado la vista `src/views/pages/entities/profile.ejs`.
- La pantalla separa los datos identificativos no editables de la informacion visible editable.
- Se ha añadido acceso directo al perfil desde la navegacion y desde el area de entidad.

### 5. Tests unitarios

- Se ha añadido un test unitario del servicio de actualizacion del perfil de entidad.
- Se ha añadido un test unitario del validador para datos invalidos.
- Ambos tests se ejecutan dentro de `npm test`.

## Migraciones y base de datos

- `RF06` no requiere cambios de esquema en Prisma.
- No ha sido necesaria ninguna migracion nueva porque los campos institucionales ya existian en el modelo `Entity`.

## Validacion realizada

- `npm run prisma:validate` ejecutado correctamente.
- `npm test` ejecutado correctamente.
- `GET /entidad/perfil` responde `200` con sesion valida de entidad.
- `POST /entidad/perfil` responde `302` a `/entidad/perfil?updated=1`.
- Los cambios de perfil quedan persistidos correctamente en PostgreSQL.

## Comprobacion manual

1. Ejecutar `npm run dev`.
2. Iniciar sesion con una cuenta de entidad.
3. Abrir `http://localhost:3000/entidad/perfil`.
4. Cambiar los datos visibles permitidos.
5. Guardar el formulario.
6. Verificar la redireccion a `?updated=1`.
7. Comprobar que el area de entidad refleja la informacion actualizada.

## Evidencia de cierre

- Flujo HTTP validado en local con una entidad temporal asociada a un usuario de prueba.
- La entidad de prueba pudo abrir el perfil y guardar cambios correctamente.
- Los datos quedaron actualizados en PostgreSQL y los registros temporales de prueba fueron eliminados tras la comprobacion.
