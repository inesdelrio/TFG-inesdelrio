# RF01 - Registro de voluntario

## Estado del requisito

Cerrado definitivamente y validado en entorno local.

## Objetivo

Permitir que un usuario no autenticado cree una cuenta de voluntario, quedando persistido en base de datos con password hasheada y rol `VOLUNTARIO`.

## Campos del formulario

- `firstName`: nombre del voluntario, obligatorio.
- `lastName`: apellidos del voluntario, obligatorio.
- `email`: email unico para acceso posterior, obligatorio.
- `phone`: telefono de contacto, opcional.
- `password`: contraseña de acceso, obligatoria con minimo de 8 caracteres.
- `passwordConfirm`: confirmacion de contraseña, obligatoria.

## Implementacion realizada

### 1. Modelo de datos

- Se ha ampliado `prisma/schema.prisma` con el enum `UserRole`.
- Se ha creado el modelo `User`.
- El modelo guarda `passwordHash` y no almacena la contraseña en texto plano.
- El rol por defecto del registro es `VOLUNTARIO`.
- El email queda protegido con restriccion `unique`.

### 2. Migracion aplicada

- Se ha generado y aplicado la migracion `20260418111532_add_user_model_rf01`.
- La migracion crea el enum `UserRole` y la tabla `User`.

### 3. Backend del registro

- Se ha creado la ruta `GET /registro`.
- Se ha creado la ruta `POST /registro`.
- Se ha implementado validacion de servidor para nombre, apellidos, email, telefono y contraseñas.
- Se comprueba duplicidad por email antes de persistir.
- La contraseña se hashea con `bcrypt` antes de guardar.

### 4. Interfaz

- Se ha creado la vista `src/views/pages/auth/register.ejs`.
- Se han añadido estilos para el formulario y mensajes de feedback.
- La navegacion principal ya enlaza al registro.

### 5. Tests unitarios

- Se ha añadido un test unitario del registro correcto en el servicio.
- Se ha añadido un test unitario de datos invalidos en la validacion del formulario.
- Los tests se ejecutan con `npm test`.

## Validacion realizada

- `npm test` ejecutado correctamente.
- `npm run prisma:validate` ejecutado correctamente.
- La app responde en `GET /registro`.
- Un `POST /registro` con datos validos devuelve redireccion a `/registro?registered=1`.
- El usuario registrado queda persistido en PostgreSQL con rol `VOLUNTARIO`.

## Comprobacion manual

1. Ejecutar `npm run dev`.
2. Abrir `http://localhost:3000/registro`.
3. Completar el formulario con un email no existente.
4. Enviar el formulario.
5. Verificar la redireccion a `/registro?registered=1`.
6. Verificar que aparece el mensaje de cuenta creada.
7. Repetir con un email ya usado y comprobar que aparece error de duplicidad.
8. Repetir con datos invalidos y comprobar mensajes de validacion.
