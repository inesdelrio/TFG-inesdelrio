# RF07 - Publicacion de eventos

## Estado del requisito

Cerrado y validado en entorno local.

## Objetivo

Permitir que una entidad verificada publique actividades o eventos con sus datos basicos y que estos aparezcan en el listado publico de la plataforma.

## Alcance funcional

- Existe formulario de publicacion en `GET /eventos/nuevo`.
- La publicacion se procesa en `POST /eventos`.
- Solo las entidades con `validationStatus = VERIFICADA` pueden publicar.
- Los eventos publicados aparecen en `GET /eventos`.
- Cada evento queda vinculado a la entidad autora.

## Implementacion realizada

### 1. Modelo de datos

- Se ha añadido el modelo `Event` en Prisma.
- Cada evento queda relacionado con `Entity` mediante `entityId`.
- Se ha añadido la relacion inversa `events` en `Entity`.
- Campos implementados:
  - `title`
  - `description`
  - `city`
  - `address`
  - `startsAt`
  - `totalSlots`

### 2. Migracion aplicada

- Se ha generado y aplicado la migracion `20260418162951_add_event_model_rf07`.
- La migracion crea la tabla `Event` y su clave foranea hacia `Entity`.

### 3. Publicacion protegida

- Se ha creado el router `src/routes/events/index.js`.
- Las rutas de publicacion requieren sesion activa y rol `ENTIDAD`.
- El servicio reutiliza `assertEntityCanPublishEvents` para bloquear entidades no verificadas.
- Si la entidad no esta verificada, la interfaz devuelve `403` y explica el motivo del bloqueo.

### 4. Validacion y persistencia

- Se ha creado el validador `src/validators/events/event.validator.js`.
- Se valida titulo, descripcion, ciudad, direccion, fecha, hora y numero de plazas.
- La fecha/hora del evento debe quedar en el futuro.
- El servicio `src/services/events/create-event.service.js` persiste el evento en PostgreSQL.

### 5. Interfaz

- Se ha creado la vista `src/views/pages/events/create.ejs` para el alta de evento.
- Se ha creado la vista `src/views/pages/events/index.ejs` para el listado publico.
- La navegacion y el area de entidad ya enlazan al flujo de publicacion.

### 6. Tests unitarios

- Se ha añadido un test unitario de creacion correcta del evento.
- Se ha añadido un test unitario de bloqueo a entidades no verificadas.
- Se ha añadido un test unitario del validador para datos invalidos.

## Validacion realizada

- `npm run prisma:validate` ejecutado correctamente.
- `npm test` ejecutado correctamente.
- `GET /eventos/nuevo` responde `200` con sesion de entidad verificada.
- `POST /eventos` responde `302` a `/eventos?created=1`.
- `GET /eventos` muestra el evento publicado en el listado publico.
- Las entidades no verificadas quedan bloqueadas para publicar.

## Comprobacion manual

1. Ejecutar `npm run dev`.
2. Iniciar sesion con una cuenta de entidad verificada.
3. Abrir `http://localhost:3000/eventos/nuevo`.
4. Completar el formulario de publicacion.
5. Guardar el evento.
6. Verificar la redireccion a `/eventos?created=1`.
7. Confirmar que el evento aparece en el listado publico.

## Evidencia de cierre

- Flujo HTTP validado en local con una entidad temporal verificada.
- Se ha publicado un evento de prueba y se ha verificado su aparicion en `GET /eventos`.
- La restriccion a entidades no verificadas sigue cubierta por tests unitarios y por la comprobacion de estado en el servicio.
