# RF11 - Suscripcion a entidades

## Estado del requisito

Cerrado y validado en entorno local.

## Objetivo

Permitir que un voluntario siga una entidad para dejar registrada la relacion y preparar los futuros avisos y notificaciones asociados a esa entidad.

## Alcance funcional

- Existe relacion `voluntario-entidad` en base de datos.
- La suscripcion se realiza desde la ficha publica del evento.
- Solo usuarios con rol `VOLUNTARIO` pueden suscribirse.
- No se permiten suscripciones duplicadas.
- La interfaz refleja si la entidad ya esta seguida.
- El voluntario puede dejar de seguir una entidad previamente seguida.

## Implementacion realizada

### 1. Modelo de datos

- Se ha creado el modelo `EntitySubscription` en Prisma.
- La relacion une `User` con `Entity`.
- Se ha añadido restriccion `@@unique([volunteerUserId, entityId])`.
- Se han añadido relaciones auxiliares en `User` y `Entity`.

### 2. Migracion aplicada

- Se ha generado y aplicado la migracion `20260420115951_add_entity_subscription_rf11`.
- La migracion crea la tabla de suscripciones y la restriccion unique contra duplicados.

### 3. Suscripcion y control de duplicados

- Se ha creado el servicio `src/services/entities/create-entity-subscription.service.js`.
- El servicio comprueba que la entidad existe antes de crear la relacion.
- Si ya existe suscripcion, devuelve error controlado.

### 4. Interfaz y ruta

- Se ha creado la ruta `POST /entidades/:entityId/suscribirse`.
- La ruta requiere sesion activa y rol `VOLUNTARIO`.
- Se ha añadido el boton `Seguir entidad` en `src/views/pages/events/detail.ejs`.
- Si la entidad ya esta seguida, la interfaz muestra el estado correspondiente en lugar del boton.

### 5. Estado de suscripcion

- Se ha creado `src/services/entities/get-entity-subscription-status.service.js`.
- El detalle del evento consulta si el voluntario ya sigue a la entidad organizadora.
- La respuesta de la interfaz cambia entre:
  - boton de suscripcion
  - estado `Ya sigues a esta entidad`

### 6. Tests unitarios

- Se ha añadido un test unitario de alta correcta de suscripcion.
- Se ha añadido un test unitario de bloqueo de suscripcion duplicada.
- Se ha añadido un test unitario de consulta del estado de suscripcion.

## Validacion realizada

- `npm run prisma:validate` ejecutado correctamente.
- `npm test` ejecutado correctamente.
- `POST /entidades/:entityId/suscribirse` crea la suscripcion correctamente.
- Una segunda suscripcion sobre la misma entidad no crea duplicados.
- La ficha del evento refleja si la entidad ya esta seguida.

## Comprobacion manual

1. Ejecutar `npm run dev`.
2. Iniciar sesion con una cuenta de voluntario.
3. Abrir la ficha de un evento publicado por una entidad.
4. Pulsar `Seguir entidad`.
5. Verificar la redireccion al detalle del evento con confirmacion.
6. Recargar la ficha y comprobar que ya no aparece el boton, sino el estado de entidad seguida.

## Evidencia de cierre

- Flujo HTTP validado en local con un voluntario temporal y una entidad temporal.
- La suscripcion se ha guardado correctamente en PostgreSQL.
- El intento repetido no genera un segundo registro y la interfaz informa del estado existente.

## Ampliacion - Dejar de seguir entidad

- Se ha creado el servicio `src/services/entities/delete-entity-subscription.service.js`.
- La baja elimina la relacion `EntitySubscription` existente, porque el modelo actual no tiene estado ni historico de suscripciones.
- Si no existe una suscripcion previa, la accion termina de forma controlada sin romper la peticion.
- Se ha creado la ruta `POST /entidades/:entityId/dejar-de-seguir`.
- En el detalle del evento, el voluntario ve `Seguir entidad` si no sigue a la entidad y `Dejar de seguir` si ya la sigue.
- Se han añadido tests unitarios para baja de suscripcion existente y baja sin suscripcion previa.
- Comprobacion manual adicional: despues de seguir una entidad desde el detalle de un evento, pulsar `Dejar de seguir` y verificar que vuelve a aparecer la accion `Seguir entidad`.
